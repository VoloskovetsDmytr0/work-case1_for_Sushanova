# 1. Словник базових англійських термінів (CLI та команди)

| English Term | Переклад | Пояснення |
|--------------|----------|-----------|
| Command | Команда | Інструкція, яку користувач вводить у терміналі |
| Command line | Командний рядок | Рядок для введення команд |
| CLI (Command Line Interface) | Інтерфейс командного рядка | Спосіб взаємодії з ОС через текстові команди |
| Shell | Оболонка | Програма-інтерпретатор команд |
| Argument | Аргумент | Дані, що передаються команді |
| Option | Опція | Додатковий параметр команди (-l, --help) |
| Flag | Прапорець | Опція без додаткового значення |
| Path | Шлях | Адреса до файлу або каталогу |
| Absolute path | Абсолютний шлях | Повний шлях від кореня системи |
| Relative path | Відносний шлях | Шлях від поточного каталогу |
| Root | Суперкористувач | Адміністратор системи |
| Process | Процес | Виконувана програма |
| Permission | Дозвіл | Права доступу до файлів |
| Environment variable | Змінна середовища | Системна змінна, що впливає на роботу програм |

# 2. Визначення понять

## CLI-режим
CLI (Command Line Interface) — режим роботи ОС, у якому користувач взаємодіє з системою через текстові команди.

## Термінал на основі графічного інтерфейсу
Програма-емулятор терміналу (GNOME Terminal, Konsole), що працює в графічному середовищі та дозволяє виконувати CLI-команди.

## Віртуальний термінал
Текстова консоль Linux, яка працює незалежно від графічної оболонки.

# 4. Порівняння оболонок

| Оболонка | Характеристика |
|-----------|----------------|
| **sh (Bourne shell)** | Найстаріша, мінімальний функціонал |
| **csh (C shell)** | Синтаксис подібний до C |
| **bash** | Найпоширеніша, підтримка скриптів |
| **tcsh** | Покращена версія csh |
| **ksh (Korn shell)** | Поєднує можливості sh та csh |
| **zsh** | Розширені можливості, плагіни, гнучке налаштування |

# 10. Віртуальні консолі Linux

За замовчуванням доступно **6 текстових консолей**.

## Перемикання:
```
Ctrl + Alt + F1–F6 — текстові консолі  
Ctrl + Alt + F7 (або F1) — графічна оболонка
```

---

# 11. Графічна оболонка

Зазвичай працює на:
```
tty7 або tty1 (залежить від дистрибутива)
```

---

# 12. Декілька входів під одним користувачем

Так, можливо входити в систему декілька разів під одним ім’ям користувача:
- з різних терміналів  
- через SSH  
- у графічному режимі  

## Переваги
- Паралельна робота  
- Виконання різних задач
