2. В ході роботи одна робоча віртуальна машина може взаємодіяти з іншою. Для цього необхідно між ними розгорнути мережу. Опишіть які типи організації мережевих з’єднань підтримуються в середовищі віртуальних машин, в чому особливість кожного з них:

## 1. Трансляція мережевих адрес (NAT — Network Address Translation)

### Опис
NAT — це режим, у якому віртуальна машина використовує мережеве підключення хост-ОС для доступу до Інтернету через трансляцію IP-адрес.

### Принцип роботи
- ВМ отримує приватну IP-адресу (наприклад, 10.0.2.x)
- Весь мережевий трафік проходить через хост
- Хост виконує трансляцію адрес (NAT) і виступає як шлюз

### Особливості
- Доступ до Інтернету: ✔
- Доступ до ВМ з локальної мережі: ❌ (за замовчуванням)
- ВМ прихована від інших пристроїв мережі
- Просте налаштування (режим за замовчуванням у більшості гіпервізорів)

### Переваги
- Безпечна ізоляція ВМ
- Не потребує складної конфігурації
- Зручно для оновлення системи та встановлення пакетів (наприклад, в Ubuntu)

### Недоліки
- Обмежений доступ до ВМ ззовні
- Для вхідних підключень потрібно налаштовувати Port Forwarding

---

## 2. Мережевий міст (Bridged Adapter)

### Опис
Bridged — це режим, у якому віртуальна машина підключається безпосередньо до фізичної мережі як окремий пристрій.

### Принцип роботи
- ВМ підключається до того ж мережевого інтерфейсу, що і хост (Wi-Fi або Ethernet)
- Отримує IP-адресу від реального DHCP-сервера (роутера)
- ВМ стає повноцінним вузлом локальної мережі (LAN)

### Особливості
- Доступ до Інтернету: ✔
- Видимість у локальній мережі: ✔
- Власна IP-адреса у тій самій підмережі, що і хост

### Переваги
- Повноцінна мережева взаємодія
- Можливість підключення до ВМ через SSH, RDP, ping
- Підходить для тестування серверів і мережевих служб

### Недоліки
- Менша ізоляція та безпека
- Залежність від налаштувань фізичної мережі
- Може блокуватися корпоративними мережами або Wi-Fi

---

## 3. Віртуальний адаптер хоста (Host-Only Adapter)

### Опис
Host-Only — це ізольована віртуальна мережа між хост-ОС і віртуальними машинами без прямого доступу до Інтернету.

### Принцип роботи
- Створюється окремий віртуальний мережевий адаптер на хості
- ВМ і хост знаходяться в одній внутрішній мережі
- Інтернет за замовчуванням відсутній (може додаватися другим адаптером NAT)

### Особливості
- Доступ до Інтернету: ❌ (за замовчуванням)
- Взаємодія: Хост ↔ ВМ: ✔
- Інші пристрої локальної мережі не бачать ВМ

### Переваги
- Висока ізоляція
- Безпечне тестове середовище
- Зручно для лабораторних робіт (наприклад, мережа між Ubuntu і її клоном)

### Недоліки
- Відсутність Інтернету без додаткових налаштувань
- Обмежена взаємодія лише з хостом і ВМ

---

## 4. Внутрішня мережа (Internal Network)

### Опис
Internal Network — це повністю ізольована мережа, у якій взаємодіють лише віртуальні машини, підключені до однієї внутрішньої мережі.

### Принцип роботи
- ВМ підключаються до спільної віртуальної мережі з однаковою назвою
- Хост-ОС не має доступу до цієї мережі
- Зовнішній доступ і Інтернет відсутні

### Особливості
- Доступ до Інтернету: ❌
- Доступ хоста до ВМ: ❌
- Взаємодія між ВМ: ✔
- Повна мережева ізоляція

### Переваги
- Максимальний рівень ізоляції
- Безпечне моделювання локальних мереж
- Ідеально для тестування клієнт-серверних застосунків між кількома ВМ

### Недоліки
- Немає доступу до Інтернету
- Немає зв’язку з хост-ОС
- Потребує ручного налаштування IP-адрес у ВМ

---

# Базові команди для налаштування мережевих параметрів в Ubuntu

Нижче наведено основні команди для перегляду, налаштування та діагностики мережевих параметрів в операційній системі Ubuntu (Linux) з поясненням їх призначення.

---

## 1. Перегляд мережевих інтерфейсів

```bash
ip a
```

**Що виконує:**
- Відображає всі мережеві інтерфейси (lo, ens33, enp0s3, eth0 тощо)
- Показує IP-адреси, MAC-адреси та стан інтерфейсу (UP/DOWN)
- Дозволяє визначити активний мережевий адаптер

---

## 2. Перегляд IP-адреси системи

```bash
hostname -I
```

**Що виконує:**
- Виводить поточні IP-адреси пристрою
- Зручно для швидкої перевірки мережевих налаштувань

---

## 3. Увімкнення мережевого інтерфейсу

```bash
sudo ip link set ens33 up
```

**Що виконує:**
- Активує (вмикає) мережевий інтерфейс
- Використовується після налаштування мережі або усунення несправностей

---

## 4. Вимкнення мережевого інтерфейсу

```bash
sudo ip link set ens33 down
```

**Що виконує:**
- Деактивує (вимикає) мережевий інтерфейс
- Застосовується для перезапуску або діагностики мережі

---

## 5. Призначення статичної IP-адреси (тимчасово)

```bash
sudo ip addr add 192.168.56.10/24 dev ens33
```

**Що виконує:**
- Призначає статичну IP-адресу мережевому інтерфейсу
- `/24` — маска підмережі (255.255.255.0)
- `dev ens33` — інтерфейс, до якого застосовується адреса
- Налаштування діє до перезавантаження системи

---

## 6. Видалення IP-адреси з інтерфейсу

```bash
sudo ip addr del 192.168.56.10/24 dev ens33
```

**Що виконує:**
- Видаляє призначену IP-адресу з мережевого інтерфейсу
- Використовується при зміні конфігурації мережі

---

## 7. Перегляд таблиці маршрутизації

```bash
ip route
```

**Що виконує:**
- Відображає таблицю маршрутів системи
- Показує шлюз за замовчуванням (default gateway)
- Вказує, куди направляється мережевий трафік

---

## 8. Налаштування шлюзу за замовчуванням

```bash
sudo ip route add default via 192.168.56.1
```

**Що виконує:**
- Встановлює основний шлюз для доступу до інших мереж та Інтернету
- `via` — адреса маршрутизатора (gateway)

---

## 9. Перевірка мережевого з’єднання

```bash
ping 192.168.56.11
```

**Що виконує:**
- Перевіряє доступність іншого вузла в мережі
- Вимірює затримку (latency) та втрати пакетів
- Використовується для тестування зв’язку між ВМ і клоном

---

## 10. Перегляд відкритих портів і з’єднань

```bash
ss -tuln
```

**Що виконує:**
- Показує відкриті TCP/UDP порти
- Відображає активні мережеві служби та з’єднання

---

## 11. Постійне налаштування мережі через Netplan

Перегляд конфігураційних файлів:
```bash
ls /etc/netplan/
```

Редагування конфігурації:
```bash
sudo nano /etc/netplan/01-netcfg.yaml
```

Застосування налаштувань:
```bash
sudo netplan apply
```

**Що виконує:**
- Дозволяє задати статичну IP-адресу, шлюз і DNS на постійній основі
- Застосовує мережеву конфігурацію після редагування YAML-файлу
