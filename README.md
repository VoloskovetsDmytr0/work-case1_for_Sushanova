1. Для чого використовують Git. Основні дії та команди

Git — це розподілена система керування версіями (Version Control System), яка дозволяє:

- відстежувати зміни у файлах;
- повертатися до попередніх версій;
- працювати над проєктом командою;
- вести історію розробки;
- об’єднувати зміни від різних користувачів.

Git широко застосовується у програмуванні, DevOps та системному адмініструванні.

2. Що таке «комміт» і як він допомагає відслідковувати зміни

Комміт — це зафіксований “знімок” стану файлів у певний момент часу.
Кожен комміт містить:

- унікальний ідентифікатор (hash);

- автора;

- дату;

- повідомлення;

- список змінених файлів.

— Як комміти відслідковують зміни?

Git зберігає історію всіх коммітів, тому можна:

- побачити, що саме змінилось
- хто зробив зміну
- коли
- повернутися до попередньої версії
