# Відповіді на теоретичні питання

## 2.1. Які команди для моніторингу стану процесів ви знаєте? Як переглянути їх можливі параметри?

Основні команди моніторингу процесів у Linux:
- `ps` — відображає список процесів
- `top` — динамічний моніторинг процесів у реальному часі
- `htop` — розширений інтерактивний монітор процесів
- `atop` — розширений системний монітор
- `uptime` — показує навантаження системи
- `free` — використання пам’яті
- `watch` — періодичний запуск команд

Перегляд можливих параметрів:
```bash
man ps
ps --help
man top
top --help
````

Команда `man` відкриває довідкову сторінку з повним описом параметрів.

---

## 2.2. Чи може команда ps у реальному часі відслідковувати стан процесів?

Ні, команда `ps` не відслідковує процеси у реальному часі.
Вона показує знімок (snapshot) стану процесів на момент виконання команди.
Для реального часу використовується команда `top` або `htop`.

---

## 2.3. За якими параметрами можливе сортування процесів в команді top? Як переключатись між ними?

У команді `top` процеси можна сортувати за:

* %CPU — використання процесора
* %MEM — використання пам’яті
* PID — ідентифікатор процесу
* TIME+ — час роботи процесу
* USER — користувач
* VIRT / RES — пам’ять

Переключення сортування:

* `P` — сортування за CPU
* `M` — сортування за пам’яттю
* `T` — за часом роботи
* `N` — за PID
* `<` або `>` — зміна поля сортування

## 2.4. Які команди для завершення роботи процесів ви знаєте?

Основні команди:

* `kill PID` — завершення процесу за PID
* `kill -9 PID` — примусове завершення
* `pkill ім'я_процесу` — завершення за назвою
* `killall ім'я_процесу` — завершення всіх процесів з такою назвою
* `xkill` — завершення через графічний інтерфейс

## Як вивести вміст директорії /proc? Де вона знаходиться та для чого призначена?

Команда:

```bash
ls /proc
```
<a href="https://ibb.co/8nGjRfHh"><img src="https://i.ibb.co/rKqZLzVW/image.png" alt="image" border="0"></a><br />

Директорія `/proc` знаходиться у кореневій файловій системі (`/`).
Вона є віртуальною файловою системою, що містить інформацію про:

* процеси (каталоги з номерами PID)
* ядро системи
* використання пам’яті
* процесор
* системні параметри

Інформація в `/proc`:

* `/proc/cpuinfo` — інформація про CPU
* `/proc/meminfo` — інформація про пам’ять
* `/proc/[PID]` — дані про конкретний процес

## Які дії можна зробити в терміналі за допомогою Ctrl + C, Ctrl + D та Ctrl + Z?

* `Ctrl + C` — примусове завершення поточного процесу
* `Ctrl + D` — вихід з терміналу або завершення введення (EOF)
* `Ctrl + Z` — призупинення процесу (переведення у background suspended)

---

## Чим відрізняється фоновий процес від звичайного. Де вони використовуються?

Звичайний (foreground) процес:

* Виконується у поточному терміналі
* Блокує командний рядок

Фоновий (background) процес:

* Працює у фоновому режимі
* Не блокує термінал
* Запускається через `&`

Приклад:

```bash
sleep 100 &
```

Використання:

* серверні служби
* довгі обчислення
* завантаження файлів
* автоматизація задач

## Опишіть команди jobs, bg, fg

* `jobs` — показує список фонових та призупинених задач
* `bg` — відновлює процес у фоновому режимі
* `fg` — переводить процес на передній план
