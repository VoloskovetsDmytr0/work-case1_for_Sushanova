1. Для чого використовують Git
   
Git — це система контролю версій, яку використовують для:

Збереження історії: Можливість повернутися до будь-якої минулої версії проєкту.

Командної роботи: Одночасна робота кількох людей над одними й тими ж файлами без конфліктів.

Страхування: Якщо код зламається, його завжди можна відновити.

Основні команди Git

Команда	і що вона робить:

git init	Створює новий репозиторій (старт відстеження змін у папці).

git clone [URL]	Копіює існуючий проєкт (наприклад, з GitHub) на ваш комп'ютер.

git status	Показує поточний стан: які файли змінені, а які нові.

git add [файл]	Додає файл до черги на збереження (підготовка).

git commit -m "опис"	Остаточно зберігає зміни (робить «знімок» стану).

git push	Відправляє збережені зміни в інтернеті (на сервер).

git pull	Завантажує чужі зміни з інтернету на ваш комп'ютер.


2. Що таке «Комміт» (Commit)
   
Комміт — це зафіксований стан файлів у певний момент часу. Це своєрідна «точка збереження» (Save point), до якої завжди можна повернутися.

Як він відслідковує зміни:

Знімок системи: Кожен комміт зберігає не просто список правок, а повний «знімок» усіх файлів проєкту на момент збереження.

Хеш (Hash): Кожному комміту присвоюється унікальний ідентифікатор (набір букв і цифр), що дозволяє точно відрізнити одну версію від іншої.

Авторство та час: Завжди відомо, хто саме (ім'я автора), коли (дата) і що саме змінив (опис змін).

Conclusion

Git is a powerful tool for version control that helps organize development, track changes, and collaborate efficiently. Commits play a key role by recording every modification and creating a complete history of the project. Using Git improves reliability, productivity, and code quality.
