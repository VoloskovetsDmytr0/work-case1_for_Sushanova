## 2. Завдання для попередньої підготовки

### 2.1 Словник базових англійських термінів

| Англійський термін | Переклад | Пояснення |
|--------------------|----------|-----------|
| Command | Команда | Інструкція, введена в терміналі для виконання дії |
| Command Line | Командний рядок | Текстовий інтерфейс взаємодії з операційною системою |
| Shell | Оболонка | Програма-інтерпретатор команд |
| Bash | Оболонка Bash | Найпоширеніша оболонка Linux |
| Prompt | Рядок запрошення | Рядок перед введенням команди |
| Option / Parameter | Параметр | Додатковий ключ команди (наприклад -l) |
| Argument | Аргумент | Об’єкт, до якого застосовується команда |
| Directory | Каталог | Папка у файловій системі |
| File | Файл | Одиниця зберігання даних |
| Environment Variable | Змінна оточення | Системна змінна, що впливає на роботу оболонки |
| Alias | Псевдонім | Скорочена назва команди |
| Function | Функція | Набір команд під одним ім’ям |
| History | Історія команд | Список раніше введених команд |
| Man Pages | Довідкові сторінки | Вбудована документація Linux |

### 2.2 Визначення базових понять
* **Командний інтерпретатор:** Програма, яка приймає команди користувача, аналізує їх та передає ядру ОС для виконання.
* **Оболонка (Shell):** Текстовий інтерфейс користувача, що дозволяє взаємодіяти з ядром та іншими програмами.
* **Команда:** Конкретна програма або вбудована функція оболонки, що виконує певне завдання.

## 3. Відповіді на теоретичні питання

### 3.1 Рядок запрошення (Prompt)
Рядок запрошення надає інформацію про поточного користувача, ім'я хоста, поточний робочий каталог та рівень привілеїв (звичайний користувач `$` або root `#`).

### 3.2 Параметри та аргументи
* **Параметри (options):** змінюють режим роботи команди (наприклад, `ls -l` — довгий формат).
* **Аргументи:** вказують об'єкт дії (наприклад, `ls /etc` — переглянути вміст конкретної папки).

### 3.3 Команда `ls`
Використовується для перегляду вмісту каталогів.
* `ls -l` — детальний вивід.
* `ls -a` — включаючи приховані файли.
* `ls -R` — рекурсивний перегляд підкаталогів.



### 3.4 Команда `echo` та змінні
Команда `echo` виводить текст або значення змінних. Змінні бувають локальні (діють лише в поточному сеансі) та змінні оточення (доступні всій системі).

## 4. Хід роботи (Практична частина)

### 4.1 Робота зі змінними
Створюємо три змінні зі своїми іменами:
```bash
var_name1="Dmytro"
var_name2="Denys"
var_name3="Oleksandr_"

# Виводимо їх значення
echo $var_name1 $var_name2 $var_name3

```

### 4.2 Робота з псевдонімами (Aliases)

```bash
alias mycal1="cal 2000"
alias mycal2="cal 2001"
alias mycal3="cal 2002"

```

### 4.3 Створення функції

```bash
students_report() {
  echo "--- Звіт по студентах ---"
  echo "Ім'я: $var_name1 | Рік: 2000"
  echo "Ім'я: $var_name2 | Рік: 2001"
  echo "Ім'я: $var_name3 | Рік: 2002"
}
# Виклик функції
students_report

```

### 4.4 Використання циклу (Control Statements)

```bash
students=("Дмитро" "Денис" "Олександр")
years=(2000 2001 2002)

for i in ${!students[@]}
do
  echo "Студент: ${students[$i]}, Рік народження: ${years[$i]}"
done

```

## 5. Контрольні запитання

1. **Типи команд у Bash:** Вбудовані (built-ins), зовнішні файли (external programs), аліаси та функції.
2. **Змінні оточення:** Системні змінні, що доступні всім процесам. Приклади: `PATH`, `HOME`, `USER`. Переглянути: `env`.
3. **Змінна `$PS1`:** Визначає формат рядка запрошення (prompt). Наприклад: `\u@\h:\w\$`.
4. **Лапки у Bash:**
* `" "` (подвійні) — дозволяють інтерпретацію змінних.
* `' '` (одинарні) — сприймають все як звичайний текст.


5. **Символи $ та #:** `$`— звичайний юзер;`#` — адміністратор (root).
6. **`whereis` vs `locate`:** `whereis` шукає шлях до двійкового файлу та man-сторінок; `locate` шукає будь-які файли по всій системі через базу даних (швидше).
